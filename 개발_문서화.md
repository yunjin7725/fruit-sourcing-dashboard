# 🍎 과일 소싱 및 마진계산기 (Premium Sourcing Dashboard) 개발 및 처리 내역 문서

## 1. 프로젝트 개요 및 목적
**과일 소싱 및 마진계산기** 프로젝트는 일반적인 B2B 도매처(econfarm, mgb2bmall 등)의 공급 단가와 정부가 제공하는 공공데이터(aT KAMIS)의 가락/지방 도매시장 시세를 비교하여 최적의 마진과 소싱 타이밍을 사용자(CEO 및 실무자)에게 제공하는 통합 대시보드입니다.

## 2. 주요 개발 및 처리 사항

### 2.1. 데이터 수집(Scraping) 및 API 연동
- **B2B 사이트 단위 단가 크롤링**
  - 목표: `econfarm`, `mgb2bmall`, `hwanggs3` 등의 기존 거래/조회 사이트에서 판매 중인 공급가를 추출.
  - 결과: 각 도매처별 정보를 스크래핑한 뒤 로컬 `csv` 파일(`econfarm_prices.csv` 등) 형태로 적재 관리합니다.
- **aT KAMIS (한국농수산식품유통공사) 오픈 API 연동**
  - 원가(Cost Basis) 산정의 기초가 되는 **'중도매인 판매가격'** 위주로 품목코드와 품종코드를 구조화했습니다.
  - 실시간으로 네트워크를 통해 API 데이터를 끌어와 최신 시세를 반영하는 `periodProductList` 메서드를 모듈화했습니다.

### 2.2. 백엔드(Flask / Python) 아키텍처 구현
- **API 엔드포인트 RESTful 설계 (`app.py`)**
  - `GET /api/fruits`: 품목 및 카테고리(딸기, 참외, 사과 등)를 코드별로 중복 없이 분류 및 리스트화.
  - `GET /api/price_trend`: 선택한 과일에 대해 현재연도부터 과거 3년 전까지의 각 물가 동향(Trend)을 계산하여 반환.
  - `GET /api/season`: 한국 시장의 실제 계절/출하시기를 반영한 `FRUIT_SEASONS` 딕셔너리 로직을 구축하여 '준비해야 할 과일', '판매 중인 과일', '마감 임박 과일'을 알고리즘으로 분리 제공.
  - `GET /api/b2b_prices`: 크롤링된 도매처 CSV 정보를 호출 시 프론트엔드로 브로드캐스트합니다.
- **데이터 전처리 시스템 구현 (`process.py`, `process_detailed.py`)**
  - 복잡한 엑셀(CSV)이나 API에서 내려온 불규칙한 딕셔너리 데이터를 판다스(Pandas)를 이용하여 정제(Cleansing)합니다. (`fruit_price_comparison.csv` 등 생성)

### 2.3. 백엔드 서버의 '4중 안정성(Stability)' 강화 조치
사용자가 **"연결이 거부됨"**과 같은 치명적인 장애를 겪지 않고 장기간 원활히 운영할 수 있도록 강력한 조치들을 취했습니다.
1. **Waitress 프로덕션 WSGI 교체**: 개발 스레드가 아닌 운영 환경에 적합한 멀티스레드 기반 Waitress 엔진으로 구동되게 변경. 
2. **0.0.0.0 글로벌 바인딩 개방**: Localhost가 아닌 모든 포트 바인딩을 열어 동일 WiFi 내 휴대폰, 타 태블릿 기기 등에서도 자유롭게 백엔드에 액세스할 수 있도록 처리.
3. **글로벌 예외 처리 방벽(Exception Handler)**: 특이한 데이터가 전송되거나 API 한도에 부딪혔을 때 서버가 크래시(Crash) 되는 것을 방지하는 `try...except` 및 `@app.errorhandler` 구성.
4. **오토-리스타트(Auto-Restarter) 배치 작성**: `start.bat`을 통한 가동 시, 시스템 오류가 강제로 발생하더라도 3초 대기 후 곧장 루프를 돌아 '자동 부활'하는 생존 기믹 적용.

### 2.4. 프론트엔드 연계 및 대시보드
- 데이터를 보기 편하게 시각화하는 Premium Sourcing Dashboard 구성 환경을 `index.html` 등의 형태로 마련했습니다 (동적인 디자인 적용 및 각종 도표 랜더링용 HTML 제공 지원).

## 3. 핵심 디렉터리 구성 (Directory Structure)
```text
과일 소싱 및 마진계산기/
 ├── app.py                      # (핵심) 대시보드 서버를 통제하는 메인 백엔드 라우터 엔진
 ├── start.bat                   # 서버 무중단 구동 및 24시간 반복 시작용 배치 스크립트
 ├── econfarm_prices.csv 등        # 수집된 B2B 도매상 기본 단가 DB 모델
 ├── fruit_price_*.csv           # KAMIS / 도매 정보가 결합된 전처리 완료 CSV 데이터
 ├── process.py (외 다수)         # 원시 데이터 파싱 및 정제용 파이썬 스루풋 스크립트
 ├── templates/, static/         # 대시보드 시각화 및 웹 페이지 레이아웃 세트
 └── .venv/                      # 고립되고 안전한 Python 가상 패키지 환경
```

## 4. 실사용 및 유지보수 가이드라인
1. **API Key 관련**: `app.py` 내부의 `KAMIS_URL`, `CERT_KEY`가 혹시 한도를 초과하거나 중지되면 정부 사이트에 재등록 후 키만 살짝 변경하시면 즉각 재활성화 됩니다.
2. **출하 일정 변경**: 기후 변화 등으로 딸기나 배의 출하시기가 바뀌었을 시, `app.py` 상단부에 선언된 `FRUIT_SEASONS` 의 `start`와 `end` 값(Month 숫자)만 교체하면 즉각 시스템에 모두 반영됩니다.
3. **크롤기 유지 관리**: `mgb2bmall`이나 `econfarm` 웹사이트의 로그인 창이나 구조가 개편될 경우 한 번씩 CSS 태그 조정 유지보수가 필요합니다.
