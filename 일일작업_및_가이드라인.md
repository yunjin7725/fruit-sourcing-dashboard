# 📝 일일 작업 정리 및 향후 가이드라인 (오늘의 작업일지)

## 📌 1. 오늘 진행한 주요 작업 요약
오늘 하루 동안 **과일 소싱 및 마진계산기 (Premium Sourcing Dashboard)** 프로젝트를 구축하고 고도화하는 과정을 거쳤습니다.

- **B2B 몰 크롤링 & 데이터 자원 확보**
  - `econfarm`, `mgb2bmall` 등 외부 도매 사이트에 로그인하여 과일 상품들의 가격을 파싱(Parsing)하고 전처리하여 `CSV` 형식으로 적재 완료.
- **KAMIS API 연동 (도매시장 공공시세 수집 시각화)**
  - 한국농수산식품유통공사의 오픈 API를 활용해 시세 데이터를 실시간 요청.
  - 테스트 및 디버깅을 거쳐, 중도매인 기준 코드를 식별하여 현재 연도부터 과거 3개년 치 **가격 변동 추세(Trend)**를 분석하는 통신 기능 완비.
- **통합 마진 산출용 백엔드 구축 (`app.py`)**
  - 수집된 도매처 CSV 데이터와 실시간 통신되는 KAMIS API 통계를 병합해 프론트엔드로 통신하는 통합 라우터(Router) 설계.
  - 한국 시장 출하 시기(Seasonality) 알고리즘 확립 및 대시보드 내 실시간 필터 전송.
- **프로덕션 급 백엔드 안전성 4중 체인 구축**
  - **Waitress WSGI 엔진**을 도입해 개발 모드가 아닌 멀티스레드 기반 강력한 분산 처리 적용.
  - **Host: 0.0.0.0 개방**을 통해 외부 내부망 어디서 접속하든 Connection Refused가 없도록 해결 및 모바일망 접근 루트 마련.
  - **글로벌 보호 처리(`@app.errorhandler`)** 및 **`start.bat` 기반 '실패 시 무한 재시작' 자동 복구 시스템** 완벽 구현.
- **불필요한 테스트 쓰레기 파일 정리 (정리 정돈)**
  - API 구조를 파악하며 임시 생성된 `temp.json`, `test_kamis.py` 등을 비롯해 수많은 연습용, 중간 과정 파일들을 모두 일괄 삭제해 핵심 파일들만 남기고 쾌적한 작업 공간 확보.

## 🚀 2. 향후 가이드라인 및 나아가야 할 길 (To-Do / Roadmap)

현재 시스템은 로컬 환경에서 매우 안정적으로 동작하지만, 실무에 더 효율적으로 도입하기 위해 다음과 같은 추가 절차를 나중에 진행하실 수 있습니다.

### [Phase 1] 데이터 유지보수 (Data Maintenance)
1. **크롤링 스크립트 주기적 재가동 규칙 수립**
   - 도매상 B2B 사이트의 가격은 시즌, 날씨, 공급량에 따라 변동될 수 있습니다. 정기적으로 크롤러 스크립트를 재구동하여 `.csv`를 최신화하는 자동 동작(예: Windows 작업 스케줄러) 연동이 추후 필요할 수 있습니다.
   - *팁: 모니터링 중 사이트 웹 디자인(UI)이 전면 개편되면 크롤러의 인식 위치가 어긋날 수 있으니 재차 HTML 수정을 요청하시면 됩니다.*
2. **KAMIS API 콜(Call) 제한 대비**
   - 현재 정부 API 키는 발급받으신 고유 계정으로 연결되어 있습니다. 트래픽 과부하로 일일 트래픽 한도(1일 콜 제한)에 걸릴 경우 데이터 표출이 중단될 수 있으므로, 에러 발생 시 곧장 새로운 키로 갱신하시면 유지보수가 수월합니다.

### [Phase 2] 프론트엔드/기능 고도화 (UI/UX 추가 구상)
1. **커스텀 부대비용 필드 적용**
   - 단순히 도매가만 보여주는 것을 넘어 *"(예상 판매가 - 도매가 단가 - 박스 포장비용 - 로켓그로스 등 물류비용 = 최종 순수익 및 마진율)"*을 즉각 계산할 수 있는 사용자 입력 및 계산기 폼 확장 구상이 도움이 됩니다.
2. **추가 도매처 확대 적용**
   - 3개에 그치지 않고, 가락시장이나 지역 공판장의 PDF 리포트 파일, 혹은 추가 폐쇄몰 데이터를 시스템에 탑재할 수 있도록 수집루트를 확대해 보십시오.

### [Phase 3] 웹 배포 (Cloud Deployment) - 최종 단계
- **로컬 PC(본인 컴퓨터)가 아닌 클라우드 서버 이관**
  - 현재는 사장님(본인) PC가 켜져 있고 `start.bat`이 동작 중이어야 휴대폰으로 조회가 가능합니다.
  - 향후 시스템이 고도화되어 타 직원과 실시간 공용 사용을 하시려거든 **AWS EC2, Cafe24 파이썬 호스팅, 혹은 Vercel/PythonAnywhere** 같은 24시간 도는 클라우드 서버로 플랫폼 단위를 승격시켜 진정한 온라인 SaaS형 사내 서비스로 진화시키기를 적극 권장합니다.
